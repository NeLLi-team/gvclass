# GVClass v1.4.0

## Summary

- Added a novelty-aware completeness estimator alongside the legacy order-baseline score.
- Refreshed the runtime resource bundle with the trained contamination model and the novelty-aware completeness resources.
- Added support diagnostics and OOD flags to the summary outputs.
- Kept the GVClass software version at `v1.4.0` while advancing the published runtime resources.

## Details

- New CLI/config option: `--completeness-mode legacy|novelty-aware`
- New output fields include:
  - `order_completeness_v2`
  - `order_completeness_v2_support_score`
  - `order_completeness_v2_ood_flag`
  - `order_completeness_v2_reference_group`
  - `order_completeness_v2_validation_mode`
  - `estimated_completeness`
  - `estimated_completeness_strategy`
- Novelty-aware completeness uses:
  - fixed family/order tier definitions
  - family-normalized strategy-2 scoring
  - offline-trained strategy-3 models
  - shrinkage for low-support novel genomes
- `estimated_contamination` now uses the shipped trained contamination model bundle (`hist_gbm_v1`) rather than the rule-based precursor score.
- Benchmark/reference workdirs were separated into the private `benchmarking/` workspace so `resources/` stays production-focused.

## Verification

- Verified the novelty-aware path on the bundled `example/` dataset with `--threads 1 -j 1 --threads-per-worker 1 --completeness-mode novelty-aware`.
- Confirmed the summary outputs include the completeness fields, contamination fields, and non-empty model-based estimates.
- Verified a fresh-start install from the published `resources_v1_5_0.tar.gz` archive on Zenodo completes and runs the example dataset successfully.
- Verified the Sylabs image pull/run path works with the updated container contents.

## Notes

- The GVClass software version remains `v1.4.0`.
- The published runtime resource bundle is now `v1.5.0`.
