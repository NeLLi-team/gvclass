# GVClass v1.4.1

## Summary

- Enabled sensitive pyhmmer filtering (`E=1e-5` / `domE=1e-5`) as the default
  runtime mode in the shipped configuration and deployment templates.
- Fixed duplicate marker counts by collapsing repeated hits from the same
  protein to the same marker model before writing filtered HMM outputs.
- Corrected downstream marker summaries so GVOG and related totals reflect the
  deduplicated per-protein-per-model counts.

## Details

- Updated the default software configuration to run in sensitive mode unless a
  run-specific config explicitly sets `sensitive_mode: false`.
- Updated the CLI fallback configuration and Prefect deployment presets to
  match the new sensitive-mode default.
- Corrected both HMM count-generation paths so repeated `(protein_id, model)`
  rows are counted once.
- Added regression tests covering:
  - per-category deduplication for marker summary metrics
  - duplicate `(protein_id, model)` collapse in filtered HMM outputs
  - deduplicated model count generation from filtered HMM output files
- Updated documentation, static version badges, and container image defaults to
  `v1.4.1`.

## Verification

- Ran the full test suite with `pixi run pytest` successfully.
- Re-ran the standard three-bin workflow and confirmed the corrected MCP and
  category-level summary values.
- Re-ran the sensitive three-bin workflow and confirmed corrected GVOG totals
  after duplicate same-protein same-model hits were collapsed.

## Notes

- The GVClass software release is `v1.4.1`.
- The published runtime resource bundle remains `v1.4.0`.
