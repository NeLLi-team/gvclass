# GVClass v1.4.2

## Summary

- Recalibrated the contamination estimate with a new `extra_trees` model trained on the new real-contig benchmark subset.
- Moved the shipped contamination model into `src/bundled_models/` so contamination-model updates no longer require a new runtime `resources/` archive.
- Kept local benchmark and codexloop artifacts out of GitHub while preserving them in local checkouts.

## Details

- The runtime now prefers `src/bundled_models/contamination_model.joblib` over `resources/contamination_model.joblib`.
- `resources/` no longer treats the contamination model as a required database file; the large novelty-aware completeness bundle remains in `resources/`.
- The bundled contamination model is now `extra_trees` with threshold `3.0`.
- Local benchmark docs (`benchmarking/README.md`, `benchmarking/RUNBOOK.md`) are preserved on disk but removed from Git tracking.
- Local-only files such as `.codexloop/`, `MEMORY.md`, and the ad hoc benchmark driver/test are ignored.
- Container/version surfaces were updated to `v1.4.2`.

## Validation

- Verified the contamination scorer loads the bundled source-tree model and reports `model_name = extra_trees`.
- Ran `pixi run pytest tests/test_real_contig_contamination_benchmark.py tests/test_database_manager.py`.
- On the completed 50-query benchmark subset:
  - clean mean dropped from `19.78` to `0.61`
  - clean median dropped from `21.73` to `0.04`
  - clean max dropped from `25.53` to `7.19`
  - contaminated mean rose from `18.92` to `39.90` versus true contaminated mean `42.16`

## Notes

- The GVClass software release is `v1.4.2`.
- The runtime completeness resource bundle remains `v1.4.0`.
- The full 70-query benchmark run was stopped after 50 completed queries because the recalibration signal was already decisive; the remaining queries are resumable from the local benchmark workspace if needed.
