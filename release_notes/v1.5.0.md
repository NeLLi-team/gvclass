# GVClass v1.5.0

## Summary

- Promoted the trained contamination model to the default production estimator.
- Bundled the contamination model together with the novelty-aware completeness resources in the runtime database.
- Split heavyweight benchmark/reference workdirs into a dedicated `benchmarking/` workspace.
- Prepared local runtime and benchmarking archives for reproducible release packaging.

## Details

- `estimated_contamination` now uses the shipped trained bundle (`hist_gbm_v1`) by default.
- Rule-based contamination fields remain in the output as diagnostics and model features.
- Database validation now requires:
  - novelty-aware completeness resources
  - the trained contamination model bundle
- The CLI preserves the modern final summary schema, including:
  - `taxonomy_strict`
  - completeness v2 fields
  - contamination fields
  - confidence fields
- Benchmarking scripts now default to the separated `benchmarking/` layout:
  - `benchmarking/contamination/...`
  - `benchmarking/completeness/...`

## Packaging

- Runtime resources version bumped to `v1.5.0` in `resources/DB_VERSION`.
- New local runtime archive target: `resources_v1_5_0.tar.gz`
- New local private benchmark archive target: `benchmarking_v1_5_0.tar.gz`

## Validation

- Verified local test suite passes after the contamination/completeness/runtime changes.
- Verified fresh example runs for:
  - `example_results`
  - `example_results_novelty`
  - `example_results_novelty_fullcheck`
- Verified regenerated example summaries include the full 69-column schema and `estimated_contamination_strategy = hist_gbm_v1`.
